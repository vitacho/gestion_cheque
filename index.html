Sistema de Gestión de Cheques

Objetivos

General

Como objetivo del proyecto es desarrollar e implementar una solución para la gestión de saldos e impresión de cheques

Específicos

  • Planificar la gestión de valores descontados o acreditados.
  • Construcción de la solución según los datos obtenidos.
  • • Testear la solución de todas las funcionalidades del sistema.

Alcance

Este proyecto abarca la creación de una solución para la gestión de saldos e impresión de cheques. En el cual se va a dividir en diferentes etapas:

Etapa 1 Etapa 2 Etapa 3
Identificar los requerimientos para el funcionamiento del sistema. Codificar la solución según lo obtenido anteriormente. Realizar pruebas unitarias de cada una de las funcionalidades.
Identificar los obstáculos que se pueden obtener. Detectar los errores que se pueden presentar.
Diseñar la planificación de la solución. Realizar la respectiva corrección de errores.

Fases

Está tecnología se divide en cinco fases que son:
  • Planificación

    • Siendo la primera fase su tarea principal es establecer metas para el proyecto y sus diferentes ciclos de iteración. El equipo se reúne con el cliente y le realiza preguntas con base en los aspectos a tomar en cuenta para el desarrollo del software requerido, con esto se toma en cuenta su visión del producto, mediante la ayuda de los desarrolladores se estima y prioriza las tareas en el plan de trabajo.

  • Diseño


    • El equipo se centra en definir las principales características del código a desarrollar, es decir la tarea principal es crear un diseño simple, ya que esto es un principio base como previamente se había mencionado, también se debe tener en cuenta los desarrolladores suelen compartir responsabilidades a esta etapa del diseño, además que cada desarrollador es responsable del diseño de una parte en concreto del código.

  • Desarrollo


    • Centrado directamente en la programación, se enfoca en producir código simple para lo cual se refactoriza de manera constante.

  • Pruebas

    • Dentro de esta metodología se debe mantener un ciclo de pruebas constante.

Justificación

  • Metodología


    Se eligió esta metodología con base en lo que nos ofrece que sería facilidad para manejar proyectos pequeños con pequeños grupos de trabajo, además que brinda una mejor compatibilidad con nuevas tecnologías gracias a que no todo el grupo de trabajo puede manejar las mismas herramientas.

    Nos ofrece también una ventaja al estar en constante comunicación, ya que esto nos provee de una alta visibilidad y responsabilidad, lo que mantiene al grupo de trabajo concentrado y comprometido. De la misma manera, esta metodología nos asegura una mejor satisfacción del cliente, por el hecho de que está involucrado de manera directa con el proyecto.

  • Django


    • Este framework se lo selecciona debido, a que utiliza Python que los integrantes del grupo de trabajo conocen, otra de las razones es debido a que Django nos permite diseñar en el mismo código las bases de datos, también a que a la parte web es fácil de montar gracias que ya tiene modelos creados, otras de las razones a su infinidad de librerías y su facilidad de integrarse en proyecto.

  • PostgresSQL


    • Se escogió esta base de datos, una de las razones es que es software libre, conjuntamente que se puede desplegar en cualquier de las plataformas tales como Windows, Linux y Mac. También es muy escalable para trabajar con altos volúmenes de trabajo y por último, ya que se puede integrar con Django y conjuntamente con la experiencia trabajando en esa tecnología.

  • Python


    • Así como la metodología XP nos ofrece un rápido desarrollo también lo hace el lenguaje de programación Python y esto nos permite un prototipado rápido, esto también nos beneficia debido al tiempo que poseemos para realizar el proyecto.
